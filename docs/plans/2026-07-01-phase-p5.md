# Codebase Atlas Phase 5 (Onboarding Tour) — Task Plan

Generated: 2026-07-01
AC coverage: US-018..US-021
NFRs in scope: COST-004, FRESH-001 (reused from P3), USABILITY-004
Estimated waves: 5
Parallelizable tasks: limited (solo, sequential) — dependency-ordered

P5 is THIN (SPEC §8: "reuses everything above"). Depends on P1 index (REQUIRED), P3 reasons
(REQUIRED), P4 representations (OPTIONAL — `representationHint` degrades gracefully to
`reasons` if P4 absent per F11). No new backend; `tour.json` lives alongside `graph.json`.

**Cross-phase contract (binding, applied here):**
- §3: `TabId` imported from P3's `src/components/PaneTabs.tsx`; P5's local `RepresentationHint`
  enum is removed (F5). `Task 1` reflects this.
- §4: `TourStep.reasonRef` is the P3 `TradeoffMatrix.id` (stable slug). No P5-local id scheme.
- §5: `checkTourStaleness` delegates to P3's `checkStaleness` from `src/core/staleness.ts` —
  NOT `freshness` / `reasonsFreshness` (F3/F4). `Task 4` reflects this.
- §6: `curateTour` accepts P3's `MatrixCache` (or `Map<string, DecisionContextEntry>`)
  directly; `projectReasonsForCuration` projects to `{decisionId, bindNodeIds: string[]}` (F7).
- §21: `curateTour` stamps `tour.metadata.graphHash = currentGraphHash` (F6).
- §24: `StartTourLink` + `LookUpInIndexLink` are merged into a single `TourCrossLinks`
  component (F16). Tasks 7 + 8 are consolidated.
- §25: `TourController` is split into a Core `TourController FSM` (pure state machine, no
  React — Task 5) + a React `TourControllerHost` (Adapters_Out — Task 6) (F18).

## Anti-patterns to watch
- `TourController` / `TourControllerHost` rendering its own graph (breaks ADR-001) — Task 10 guard.
- Redefining P3 `TradeoffMatrix` or P3 `TabId` (breaks "reference, don't duplicate") — Task 1 test.
- Defining a local `RepresentationHint` enum (F5) — Task 1 imports `TabId` from P3.
- Duplicating the freshness mechanism (breaks FRESH-001 reuse) — Task 4 imports P3's
  `checkStaleness`; invariant test regex matches the actual export (F3).
- Re-curation on every launch (breaks COST-004) — Task 9 cache + Task 11 cache-gate.
- `curateTour` not stamping `graphHash` (F6) — Task 9 stamps it; Task 9 test asserts it.
- Hand-authored tour with `graphHash === undefined` reported as stale (F15) —
  `checkTourStaleness` returns `'fresh'` on undefined; F6 ensures curation always stamps it.
- Corrupt `tour.json` crashing the app (F14) — Task 11 wraps load in try/catch.

## Invariants enforced by tests
1. **TourStep references existing ids, never duplicates data** → Task 1.
2. **TourStep does NOT define a local tab enum (F5) — imports `TabId` from P3's PaneTabs** → Task 1.
3. **TourStep does NOT have a `clusterId` field (F12) — cluster-level steps deferred** → Task 1.
4. **TourMetrics reuses `AtlasNode.fanIn`/`fanOut` populated by NodeTyper, does not
   recompute (F17)** → Task 2.
5. **TourSerializer round-trips tour.json losslessly** → Task 3.
6. **Staleness reuses P3's `checkStaleness` from `src/core/staleness.ts` (no duplicate, F3/F4)**
   → Task 4.
7. **`checkTourStaleness` returns `'fresh'` when `graphHash` is undefined (F15)** → Task 4.
8. **TourController FSM (Core, no React) emits focus+tab commands; TourControllerHost
   (Adapters_Out, React) imports nothing from React Flow** → Task 5, 6, 10 (F18).
9. **`representationHint` is projected onto the registered tab set, falling back to
   `'reasons'` when the hint's tab is not registered (F11)** → Task 6.
10. **`TourCrossLinks` (single component) renders both cross-link directions (F16)** →
    Task 7+8 (consolidated).
11. **TourCuration is cached — re-curate only on missing/stale cache (COST-004)** → Task 9, 11.
12. **`curateTour` stamps `graphHash` from the current graph (F6)** → Task 9.
13. **`curateTour` accepts P3's `MatrixCache` shape (F7)** → Task 9 (+ Task 9a for
    `projectReasonsForCuration`).
14. **Deleted `nodeId` and dangling `reasonRef` degrade gracefully (F9, F10)** → Task 6/11.
15. **Mid-playback non-tour node click pauses the tour (F13)** → Task 6/11.
16. **Corrupt `tour.json` at app load degrades to no-tour mode (F14)** → Task 11.
